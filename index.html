what is stride

$ stride --describe An agent that builds the thing, not just talks about it.

Most assistants describe an app and leave you to build it. Stride does the opposite: it reads your intent, matches it to a build strategy, assembles a fully functional app, and runs it live in front of you. The output is a real artifact — a single self-contained HTML file you can download, open anywhere, and use offline.

  • ▸ 01
    One smart input. No mode maze. Type what you want in your own words; Stride routes it to the right build strategy.
  • ▸ 02
    Real, working output. Every build is interactive and complete — not a mockup, not a screenshot. It runs the moment it lands.
  • ▸ 03
    Yours to keep. Save any build as a standalone .html file. It has no dependencies and works without a network.
  • ▸ 04
    A build log. Everything you forge is kept in the session rail so you can revisit and restore earlier apps instantly.
02
how it works

$ stride --process Three moves from idea to running app.

STEP_01

Describe

Tell Stride what you need in plain language. "A pomodoro timer." "Split this bill." "A habit tracker." The router reads the intent.

STEP_02

Forge

Stride selects a build strategy and assembles a complete app — structure, logic, and a distinct interface tuned to that kind of tool.

STEP_03

Walk

The app materializes in the live pane, fully interactive. Use it, refine it, or save it as a file you own forever.
